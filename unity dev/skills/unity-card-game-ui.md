# Skill: Unity Card Game UI (Lobby / Room / Battle)

## 适用场景

用于实现和迭代客户端 UI：
- 主菜单（创建房间/加入房间）
- 房间页（玩家列表、准备状态、开始按钮）
- 对局 HUD（抽牌堆、弃牌堆、手牌、提示、回合状态）
- 结算页（亮牌、计分、排名）

## 设计来源

- Docs/GDD.md 中的核心循环与美术方向
- Proto 广播消息驱动的状态变化

## UI 架构要求

- UI 只渲染 ViewModel，不直接读写网络层
- 网络消息先更新 DomainState，再由 UI 订阅刷新
- 所有按钮状态由 phase + current_player_id 决定

## 最小界面清单

1. LobbyScene
- 输入昵称
- 创建房间
- 输入房间码并加入

2. RoomScene
- 房间码展示
- 玩家列表（昵称、座位、在线、准备）
- 准备/取消准备
- 房主开始游戏

3. GameScene
- 顶栏：轮数、当前玩家、阶段
- 中央：抽牌堆数量、弃牌堆顶牌
- 底部：自己 4 张牌槽位
- 侧边：其他玩家状态与累计分
- 操作面板：抽牌、弃牌堆替换、喊稳态

4. ResultPanel
- 本轮亮牌
- 本轮得分与累计分
- 最终排名

## 关键交互规则

- 非自己回合时禁用操作按钮
- DrawCardRsp 之后必须进入二段决策 UI
- 技能 UI 只处理 7-12 直接弃牌后的 peek_self / spy / swap；不要实现旧版 SkillPromptNotify 或 13 看+换
- 替换 UI 使用 slot_indices，支持多选槽位和失败后牌区数量超过 4 的显示
- RoundRevealNotify 到达后锁定操作，进入结算态

## 视觉与可用性建议

- 保持深色赛博医疗风，但以可读性优先
- 高风险动作（喊稳态/交换）使用二次确认
- 对 server error.message 显示用户可懂提示
- 关键状态变化提供轻量动画与音效

## 验收清单

- 房间到对局页面切换稳定
- 每个广播都能驱动 UI 更新
- 非法时机按钮不可点击
- 弱网下不会出现多次重复点击提交
