## 服务端架构

```
GameServer (TCP 入口)
    → RoomService (房间 CRUD + Ready/Start)
    → GameService (权威游戏逻辑)
        → handleDrawCard
        → handleDiscardDrawn
        → handleReplaceWithDrawn
        → handleTakeFromDiscard
        → handleUseSkill (PeekSelf / Spy / Swap)
        → handleCallSteady
        → endTurn → sendTurnStart
        → revealAndScore
```

**游戏状态机**：`WaitingToStart → Playing → WaitingDrawDecision → Playing → ... → Reveal → WaitingToStart / GameOver`

## 消息流（关键）

每条操作的消息顺序：`Rsp(私信) → ActionResultNotify(广播) → TurnStartNotify(广播)`

| 消息 | 方向 | 关键字段 |
|------|------|---------|
| GameStartNotify | 私信(每人不同) | own_cards[].isKnown/value, opponent_hands[].cardCount, scores |
| TurnStartNotify | 广播 | current_player_id, turn/round, phase, draw/discard pile |
| DrawCardRsp | 私信 | value, skill |
| DiscardDrawnRsp | 私信 | error |
| ReplaceWithDrawnRsp | 私信 | exchange_result(success, slots, value, discarded) |
| TakeFromDiscardRsp | 私信 | exchange_result |
| UseSkillRsp | 私信 | peeked_value, swap_occurred |
| ActionResultNotify | 广播 | action_type, source/target_player_id, skill_used, swap_occurred, source/target_slot, player_hands[], pile counts |
| RoundRevealNotify | 广播 | revealed_hands(所有玩家手牌值), scores(计分明细) |
| ScoreUpdateNotify | 广播 | scores(累计分) |
| GameOverNotify | 广播 | rankings |

## CLI 客户端架构

```
ClientApp::run()
  → connectToServer → loginFlow → roomFlow
  → waitingRoomLoop (非阻塞 select stdin + socket)
  → gameLoop (状态机驱动，非阻塞)
      GameSubState 枚举: IDLE → AWAITING_MAIN_INPUT
        → WAITING_DRAW_RSP → AWAITING_DRAWN_DECISION
        → WAITING_DISCARD_RSP → SKILL_* → WAITING_SKILL_RSP
        → WAITING_REPLACE_RSP / WAITING_TAKE_RSP
        → WAITING_CALL_STEADY_RSP
  → handleRoundRevealPhase → 回合间等待
```

**关键设计**：
- 单线程 + select() 非阻塞 stdin，不阻塞网络消息
- `drainMessages()` 先处理所有消息再决策
- `GameSubState` 状态机管理输入流程
- 回合间结算面板显示 ready 状态
