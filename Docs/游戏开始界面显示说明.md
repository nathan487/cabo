# 游戏开始时的界面显示说明

## 📋 概述

点击 Start Game 后，服务端会发送两个通知：
1. **GameStartNotify** - 游戏开始，包含初始状态
2. **TurnStartNotify** - 第一个回合开始，指定当前玩家

## 🎮 界面显示逻辑

### 流程时序

```
用户点击 Start Game
    ↓
服务端处理
    ↓
发送 GameStartNotify (给所有玩家)
    ↓
发送 TurnStartNotify (给所有玩家)
    ↓
客户端接收并渲染界面
```

## 🐛 潜在问题检查

### ❓ 问题1：GameStartNotify 会立即显示按钮吗？

**答案：是的！**

**代码位置：** `GameTableUIToolkit.cs:251-256`

```csharp
// GameStartNotify 处理
if (gs.FirstPlayerId == myPlayerId)
{
    ShowMainButtons();  // ← 立即显示按钮
    phaseText.text = ">>> YOUR TURN <<<";
}
```

**问题：** 这可能导致按钮显示两次（GameStartNotify + TurnStartNotify）

**建议：** GameStartNotify 可以不显示按钮，等 TurnStartNotify 来显示：

```csharp
// 修改建议
if (gs.FirstPlayerId == myPlayerId)
{
    // 先不显示按钮，等TurnStartNotify
    phaseText.text = "Waiting for turn...";
}
else
{
    HideAllButtons();
    phaseText.text = "Opponent's turn...";
}
```

### ❓ 问题2：如果只收到 GameStartNotify，没收到 TurnStartNotify 呢？

**理论情况：** 不会发生，服务端保证先后发送

**代码保证：** `GameService.cc:165-166`
```cpp
sendGameStart(room);
sendTurnStart(room);  // 立即发送
```

**但是：** 网络可能有延迟，建议在 GameStartNotify 也做判定

## 📝 总结

### ✅ 当前实现

**优点：**
- ✅ 能够正确判定当前玩家
- ✅ 当前玩家可以看到按钮并操作
- ✅ 非当前玩家看不到按钮，只能等待
- ✅ 提示文本清晰，颜色区分明显
- ✅ 有详细的调试日志

**可能的小问题：**
- ⚠️ GameStartNotify 和 TurnStartNotify 都会设置UI，可能有轻微闪烁
- ⚠️ 非当前玩家的提示显示 Player ID 数字，不够友好

### 💡 改进建议

1. **优化 GameStartNotify 处理**
   - 不要在 GameStartNotify 中显示按钮
   - 等 TurnStartNotify 统一处理

2. **改进提示文本**
   - 显示玩家昵称而不是 Player ID
   - "Opponent's turn (Alice)..." 比 "Opponent's turn (Player 10000)..." 更友好

3. **添加动画效果**
   - 按钮显示时添加淡入动画
   - 提示文本闪烁或高亮
