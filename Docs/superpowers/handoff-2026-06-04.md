## 一、项目概览

基于 Kabo 桌游改编的多人在线卡牌游戏。技术栈：
- **客户端**: Unity 2022.3.62f3c1 (C#) + protobuf
- **服务端**: C++ (Muduo 网络库) + protobuf
- **协议**: TCP 长连接, `[4字节大端长度][protobuf数据]` 帧格式

## 三、当前状态

### 3.1 服务端

**可编译运行**。启动命令:
```bash
cd MuduoBaseGameServer/build
cmake .. -DCMAKE_BUILD_TYPE=Debug && make -j$(nproc)
LD_PRELOAD=$HOME/anaconda3/lib/libprotobuf.so.31 ./GameServer
```

监听 8888 端口，房间+游戏逻辑全部就绪。

### 3.2 客户端 — 编译

Unity 项目 **零编译错误**，零警告 (已修复 `gameStarted` 未使用和 TMP 中文字体问题)。

### 3.3 客户端 — 运行时状态

**可工作:**
- MainMenuScene: 网络大厅 UI (连接/创建房间/加入/准备/开始)
- 消息收发正常，客户端能收到服务端所有通知

**有问题 (核心阻塞):**
- **GameScene 对局界面显示不正确**。Start Game 后场景加载，但 UI 渲染有问题(卡牌/按钮/对手区域位置错乱或不显示)
- **根本原因**: 当前设计在 GameScene 中通过 Editor 脚本创建基础 UI，然后在运行时通过 `GameTableUI.Initialize` → `OnGameStarted` 动态 Instantiate Prefab 创建玩家区域和卡牌。这个 Prefab 实例化流程有多个问题:
  1. Prefab 引用通过 Editor `AssetDatabase.LoadAssetAtPath` 设置 → 运行时可靠性未知
  2. `cardsContainer` 反射绑定因 public/private 变化失效
  3. 运行时创建 GameObjects 的布局位置与 Scene 已有的 HUD/Pile 区域配合不好
  4. TurnStartNotify 在场景加载前到达导致按钮永远隐藏

## 六、特殊注意事项

1. **ProtoGateway 不是 MonoBehaviour** — 是 `new` 出来的普通对象。需要用 `RoomClientController.GetGateway<ProtoGateway>()` 获取，不能用 `GetComponent`。
2. **muduo 类在全局命名空间** — 不是 `mymuduo::`，所以 `EventLoop`/`TcpServer` 等直接用。
3. **protobuf 3.20.3 via anaconda** — 头文件 `/home/niuma/anaconda3/include`，库 `/home/niuma/anaconda3/lib`。
4. **RPATH 问题** — 服务端编译后用 `patchelf` 修复，去掉 anaconda lib 路径避免 libstdc++ 版本冲突。
5. **服务端日志** — muduo 底层日志已改为 `LOG_DEBUG`（通过 Channel.cc/EPollPoller.cc 的 sed 替换）。
6. **Scene 中文字体** — 不要用 TMP 默认字体(无中文)。改用 legacy `UnityEngine.UI.Text` + `Font.CreateDynamicFontFromOSFont("Microsoft YaHei", ...)`。
7. **客户端线程安全** — TCP 收包在后台线程，通过 `incomingQueue` 入队，主线程 `TickCoroutine` 出队处理。
8. **协程启动时机** — `TickCoroutine` 在 `ProtoGateway` 构造时启动但 Status 为 Disconnected 会立即退出。已改为 `while(true)` 循环。

## 七、测试流程

1. 启动服务端:
```bash
cd "MuduoBaseGameServer/build"
LD_PRELOAD=$HOME/anaconda3/lib/libprotobuf.so.31 ./GameServer
```

2. Unity 打开项目，MainMenuScene → Play
3. 需要两个客户端: Build 一个 exe(File→Build Settings→Build)，或复制项目开两个 Editor
4. Client A: Connect → Create Room → 记下房间码 → Ready
5. Client B: Connect → Join Room(输入码) → Ready
6. Client A: Start Game → 自动跳转 GameScene

## 八、MCP 可用

Unity 已配置 unity-mcp (`com.coplaydev.unity-mcp`)，可通过 HTTP API 操作:
- `read_console` — 读控制台
- `execute_menu_item` — 执行菜单 (如 `Tools/Build Game Scene (Network)`)
- `manage_editor` — 进/出 Play 模式
- `manage_gameobject` — 增删改查 GameObject
- `execute_code` — 执行任意 C# 代码

初始化: POST `/mcp` → `initialize` → 提取 `Mcp-Session-Id` → 后续请求带 header。
