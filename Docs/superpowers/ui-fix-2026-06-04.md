# GameScene UI 修复 - 2026-06-04

## 问题描述

GameScene 对局界面在从 MainMenuScene 跳转后显示不正常：
- 卡牌、对手区域、操作按钮位置错乱或不显示
- 根本原因：运行时动态实例化 Prefab 导致布局问题和引用失效

## 解决方案

采用**预创建 UI 元素**的方案，在 Scene 中直接创建所有 UI，运行时只更新数据。

## 修改文件

### 1. GameSceneBuilder_Network.cs (重写)

**改动：**
- 移除 Prefab 引用逻辑（不再使用 `AssetDatabase.LoadAssetAtPath`）
- 新增 `CreatePlayerArea()` 方法：直接在场景中创建完整的 PlayerArea
  - 包含背景、名称文本、分数文本、卡牌容器
  - 使用 `HorizontalLayoutGroup` 自动布局 4 张卡牌
- 新增 `CreateCardView()` 方法：创建单个 CardView
  - 包含背景图片、卡背图片、数值文本、点击按钮
  - 通过反射设置私有字段
- 预创建 2 个 PlayerArea：
  - `SelfArea` (位置: y=-220) - 玩家自己
  - `OpponentArea` (位置: y=160) - 对手
- 每个 PlayerArea 包含 4 个完全初始化的 CardView

**关键代码：**
```csharp
var selfArea = CreatePlayerArea(tableRoot.transform, "SelfArea", new Vector2(0, -220), true);
var oppArea = CreatePlayerArea(tableRoot.transform, "OpponentArea", new Vector2(0, 160), false);

SetPrivateField(tableUI, "selfArea", selfArea);
SetPrivateField(tableUI, "opponentArea", oppArea);
```

### 2. GameTableUI.cs (简化)

**移除：**
- `cardViewPrefab` 和 `playerAreaPrefab` 字段
- `CreatePlayerArea()` 方法及其所有逻辑
- `CreateCard()` 方法及其所有逻辑
- `StoreArea()` 方法
- `GetSeatIndex()` / `GetOpponentSeatIndex()` 方法
- `CreateTextElement()` 辅助方法
- `playerAreas[]` 数组、`layout[]` 数组、`totalPlayers` 变量

**新增：**
- `[SerializeField] private PlayerAreaView selfArea;`
- `[SerializeField] private PlayerAreaView opponentArea;`

**改动：**
- `OnGameStarted()`：不再动态创建区域，直接初始化已存在的 `selfArea` 和 `opponentArea`
- `OnTurnStarted()`：直接操作 `selfArea` 和 `opponentArea` 的高亮状态
- `OnActionReceived()`：直接更新 `opponentArea` 的卡牌数量
- `OnReplaceResult()`：直接更新 `selfArea` 的卡牌已知状态
- `OnScoreUpdated()`：直接更新 `selfArea` 和 `opponentArea` 的分数

**核心逻辑（OnGameStarted）：**
```csharp
// Initialize self area
if (selfArea != null && myView != null)
{
    selfArea.Initialize(myPlayerId, "你", 0);
    for (int i = 0; i < 4; i++)
    {
        if (selfArea.CardViews[i] != null)
        {
            selfArea.CardViews[i].Initialize(i);
            var oc = FindOwnCard(myView, i);
            if (oc != null && oc.IsKnown)
                selfArea.CardViews[i].SetKnown(oc.Value);
            else
                selfArea.CardViews[i].SetUnknown();
        }
    }
}

// Initialize opponent area
if (opponentArea != null && myView != null && myView.OpponentHands.Count > 0)
{
    var firstOpp = myView.OpponentHands[0];
    opponentArea.Initialize(firstOpp.PlayerId, "对手", 0);
    opponentArea.SetCardCount(firstOpp.CardCount);
    
    for (int i = 0; i < 4; i++)
    {
        if (opponentArea.CardViews[i] != null)
        {
            opponentArea.CardViews[i].Initialize(i);
            opponentArea.CardViews[i].SetUnknown();
            opponentArea.CardViews[i].gameObject.SetActive(i < firstOpp.CardCount);
        }
    }
}
```

## 技术细节

### 反射设置私有字段

由于 `PlayerAreaView` 和 `CardView` 的字段是 `[SerializeField] private`，Builder 脚本使用反射设置：

```csharp
private static void SetPrivateField(object target, string fieldName, object value)
{
    var field = target.GetType().GetField(fieldName,
        System.Reflection.BindingFlags.NonPublic | System.Reflection.BindingFlags.Instance);
    if (field != null)
    {
        field.SetValue(target, value);
    }
    else
    {
        Debug.LogWarning($"[GameSceneBuilder] Field not found: {fieldName} on {target.GetType().Name}");
    }
}
```

### CardViews 数组初始化

`PlayerAreaView.CardViews` 是一个属性，需要直接设置：

```csharp
typeof(Cabo.Client.Game.PlayerAreaView)
    .GetProperty("CardViews")
    .SetValue(areaView, cardViews);
```

### 布局使用 HorizontalLayoutGroup

卡牌容器使用 Unity 的自动布局：

```csharp
var horizLayout = cardsContainer.AddComponent<HorizontalLayoutGroup>();
horizLayout.childForceExpandWidth = false;
horizLayout.childForceExpandHeight = false;
horizLayout.spacing = 10;
horizLayout.childAlignment = TextAnchor.MiddleCenter;
```

这样 4 张卡会自动水平排列，间距 10 像素。

## 优势

1. **稳定性**：UI 元素在 Scene 中预创建，不依赖运行时 Prefab 引用
2. **可预览**：直接在 Scene 视图中看到完整布局
3. **简化代码**：GameTableUI 只负责数据更新，不负责 UI 创建
4. **性能**：不需要运行时 Instantiate，场景加载即可用
5. **调试友好**：所有 UI 元素在 Hierarchy 中可见，方便调试

## 局限性

- 当前只支持 2 人对局（1 个对手）
- 如需支持 3-4 人，需要在 Builder 中预创建更多 PlayerArea
- 或者保留动态创建逻辑，但需要修复 Prefab 引用问题
