# 问题诊断：场景加载时序问题

## 🔍 问题现象

从日志看到：
1. GameTableUIToolkit 初始化
2. "No pending TurnStartNotify" - 正常
3. **"Standalone mode: showing test UI"** - 进入独立测试模式！
4. 看到一个一闪而过的 "YOUR TURN" 画面
5. 然后跳转到游戏场景

## 🐛 根本原因

### 时序问题

```
1. 用户点击 Start Game
   ↓
2. 收到 GameStartNotify
   - 保存到 GameSceneBootstrap.PendingGameStart
   - 启动协程加载场景
   ↓
3. 场景开始加载 GameScene
   ↓
4. 🚨 问题：GameSceneController.Start() 被调用
   - 找到 ProtoGateway
   - 调用 GameTableUIToolkit.Initialize(gateway, gameCtrl)
   ↓
5. GameTableUIToolkit.Initialize() 执行
   - 订阅事件
   - 检查 PendingTurnStart (No)
   - 检查 PendingGameStart (应该有！)
   ↓
6. 🚨 但是 PendingGameStart 是 null！
   - 进入 "Standalone mode: showing test UI"
   ↓
7. 场景加载完成，但已经进入测试模式
```

### 为什么 PendingGameStart 是 null？

**可能的原因：**

1. **场景重新加载导致静态变量清空**
   - `GameSceneBootstrap.PendingGameStart` 是静态变量
   - 但如果 GameScene 已经在场景列表中，重新加载会清空

2. **OnGameStarted 被调用了两次**
   - 第一次：通过 `OnStartGame?.Invoke(notify)` 在旧场景
   - 第二次：通过 `PendingGameStart` 在新场景
   - 但第二次时已经被清空

3. **场景加载时机问题**
   - ProtoGateway 在 `OnGameStartNotify` 中触发场景加载
   - 但同时也触发了 `OnStartGame?.Invoke(notify)`
   - 如果旧场景中有订阅者，会消费这个事件

## 💡 解决方案

### 方案 1：确保 PendingGameStart 不被过早清空

**问题代码：** `ProtoGateway.cs:471-479`

```csharp
private void OnGameStartNotify(GameStartNotify notify)
{
    Debug.Log($"[ProtoGateway] Game started: round={notify.RoundNumber}, firstPlayer={notify.FirstPlayerId}");
    GameSceneBootstrap.PendingGameStart = notify;  // ← 保存
    
    if (coroutineOwner != null)
        coroutineOwner.StartCoroutine(LoadGameSceneNextFrame());
    
    OnStartGame?.Invoke(notify);  // ← 🚨 这里可能触发订阅者消费
}
```

**问题：** 如果在 LobbyRoomScene 中有组件订阅了 `OnStartGame`，会导致：
1. 触发订阅者
2. 可能清空或修改 `PendingGameStart`
3. 场景加载后找不到数据

### 方案 2：不在 OnGameStartNotify 中触发事件（推荐）

**修改：** `ProtoGateway.cs`

```csharp
private void OnGameStartNotify(GameStartNotify notify)
{
    Debug.Log($"[ProtoGateway] Game started: round={notify.RoundNumber}, firstPlayer={notify.FirstPlayerId}");
    GameSceneBootstrap.PendingGameStart = notify;
    
    // Defer scene load to avoid interrupting message processing loop
    if (coroutineOwner != null)
        coroutineOwner.StartCoroutine(LoadGameSceneNextFrame());
    
    // ❌ 不要在这里触发事件，因为：
    // 1. GameTableUIToolkit 还不存在（在旧场景）
    // 2. 会导致 PendingGameStart 被过早消费
    // OnStartGame?.Invoke(notify);  // ← 注释掉
}
```

**理由：**
- GameStartNotify 应该由 GameTableUIToolkit 通过 `PendingGameStart` 处理
- 不需要通过事件系统传递
- 避免在错误的场景中触发

### 方案 3：在 GameTableUIToolkit 中先处理 GameStart 再处理 TurnStart

**当前代码顺序：**
```csharp
// 1. 订阅事件
gw.OnTurnStart += OnTurnStarted;

// 2. 检查 pending turn
var pendingTurn = gw.GetPendingTurnStart();

// 3. 检查 pending game start
var gs = GameSceneBootstrap.PendingGameStart;
```

**问题：** 顺序错误！应该先处理 GameStart，再处理 TurnStart

**修改后：**
```csharp
// 订阅游戏事件（仅在有 gateway 时）
if (gw != null)
{
    Debug.Log("[GameTableUIToolkit] Subscribing to gateway events...");
    gw.OnStartGame += OnGameStarted;
    gw.OnTurnStart += OnTurnStarted;
    // ... 其他订阅
    Debug.Log("[GameTableUIToolkit] ✅ All events subscribed");
}

// 先处理 GameStartNotify（初始化游戏状态）
var gs = GameSceneBootstrap.PendingGameStart;
if (gs != null)
{
    GameSceneBootstrap.PendingGameStart = null;
    OnGameStarted(gs);
}
else
{
    Debug.Log("[GameTableUIToolkit] Standalone mode: showing test UI");
    roundInfo.text = "UI Toolkit Test Mode";
    phaseText.text = "Font test: English 123";
}

// 再处理 TurnStartNotify（显示回合按钮）
if (gw != null)
{
    var pendingTurn = gw.GetPendingTurnStart();
    if (pendingTurn != null)
    {
        Debug.Log($"[GameTableUIToolkit] 📥 Processing cached TurnStartNotify (Turn {pendingTurn.TurnNumber})");
        OnTurnStarted(pendingTurn);
    }
    else
    {
        Debug.Log("[GameTableUIToolkit] No pending TurnStartNotify");
    }
}
```

## 🎯 推荐修复方案

**同时应用方案 2 和 方案 3：**

1. **删除 OnGameStartNotify 中的事件触发**
2. **调整 GameTableUIToolkit 的处理顺序**

这样可以确保：
- ✅ GameStartNotify 只通过 PendingGameStart 传递
- ✅ 处理顺序正确（先 GameStart，再 TurnStart）
- ✅ 避免竞态条件

public void Initialize(ProtoGateway gw, GameClientController gc)

            Debug.Log($"[GameTableUIToolkit] 📥 Processing cached TurnStartNotify (Turn {pendingTurn.TurnNumber})");
            OnTurnStarted(pendingTurn);
        }
        else
        {
            Debug.Log("[GameTableUIToolkit] No pending TurnStartNotify");
        }
    }
    
    Debug.Log("[GameTableUIToolkit] ✅ Initialize complete");
