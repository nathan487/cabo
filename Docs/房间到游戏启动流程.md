## 当前玩家判定机制

### 服务端判定逻辑

#### 数据结构（GameService.h）
```cpp
struct GameRoom {
    int32_t currentPlayerSeat = 0;  // 当前回合玩家的座位号 (0-based)
    std::vector<std::shared_ptr<PlayerGameState>> players;  // 玩家列表（按座位顺序）
};

struct PlayerGameState {
    int64_t playerId;     // 玩家唯一ID
    int32_t seatId;       // 座位号
};
```

#### 判定函数（GameService.cc:50-53）
```cpp
bool GameService::isCurrentPlayer(GameRoom& room, int64_t playerId) {
    auto p = getPlayer(room, playerId);
    return p && p->seatId == room.currentPlayerSeat;
}
```

**核心逻辑：**
- 通过 `playerId` 找到对应的 `PlayerGameState`
- 比较该玩家的 `seatId` 是否等于 `room.currentPlayerSeat`
- 相等则为当前回合玩家

#### 回合推进（GameService.cc:283-287）
```cpp
void GameService::nextPlayer(GameRoom& room) {
    do {
        room.currentPlayerSeat = (room.currentPlayerSeat + 1) % static_cast<int32_t>(room.players.size());
    } while (room.steadyCallerSeat >= 0 && room.currentPlayerSeat == room.steadyCallerSeat);
}
```

**逻辑：**
1. `currentPlayerSeat` 递增，循环回到 0
2. 如果已喊稳态（最终轮），跳过稳态喊叫者
3. 确保回合按座位顺序轮转

#### 回合通知（GameService.cc:210-234）
```cpp
void GameService::sendTurnStart(GameRoom& room) {
    room.turnNumber++;
    auto current = room.players[room.currentPlayerSeat];  // 通过座位索引获取当前玩家
    
    ::game::messages::ServerMessage msg;
    auto* ts = msg.mutable_turn_start_notify();
    ts->set_current_player_id(current->playerId);  // 发送当前玩家的 playerId
    ts->set_turn_number(room.turnNumber);
    ts->set_round_number(room.roundNumber);
    
    broadcastToRoom(room, msg);  // 广播给所有玩家
}
```

private void OnTurnStarted(TurnStartNotify ts)

1. 服务端广播 `TurnStartNotify`，包含 `current_player_id`
2. 客户端接收后保存到 `CurrentTurnPlayerId`
3. UI层比较 `ts.CurrentPlayerId == myPlayerId`
4. 是当前玩家则显示操作按钮，否则隐藏

### 4. 开始游戏（核心流程）

#### 客户端发起
```
LobbyRoomDemoUI.OnStartGameClicked()
  ↓
RoomClientController.StartGame()
  ↓
ProtoGateway.StartGame()
  ↓
构造 StartGameReq (player_id, room_id)
  ↓
发送到服务端
```

#### 服务端处理（GameServer.cc:108-134）
```
MessageDispatcher → handler (消息类型 14)
  ↓
GameServer.onStartGame(conn, msg)
  ↓
1. RoomService.handleStartGame(conn, msg)
   ├─ 验证请求者是房主
   ├─ 验证所有玩家已准备
   ├─ 发送 RoomStartNotify 给所有玩家
   └─ 返回 StartGameRsp
  ↓
2. 从 roomService 获取 Room 数据
  ↓
3. 构造 PlayerGameState 列表
   for each player in room:
     - playerId
     - nickname
     - seatId
     - conn (连接)
     - totalScore (初始为0)
  ↓
4. GameService.startGame(roomId, gamePlayers, hostPlayerId)
```

#### GameService.startGame（GameService.cc:124-137）
```
1. 创建 GameRoom 对象
   - roomId
   - hostPlayerId
   - maxPlayers
   - players (PlayerGameState 列表)
   - step = GameStep::Playing
  ↓
2. games_[roomId] = room (保存到游戏映射表)
  ↓
3. startNewRound(room)
```

#### GameService.startNewRound（GameService.cc:139-167）
```
1. 初始化回合状态
   - roundNumber++
   - currentPlayerSeat = 0  ← 第一个玩家（座位0）开始
   - turnNumber = 0
   - steadyCallerSeat = -1
  ↓
2. initDeck(room) → 洗牌
   - 清空抽牌堆和弃牌堆
   - 添加所有卡牌 (0×2, 1-12×4, 13×2)
   - 分配技能类型
   - std::shuffle 洗牌
  ↓
3. 发4张牌给每位玩家
   for each player:
     - cards.clear()
     - knownSlots.clear()
     - 抽4张牌到 player->cards
     - 标记前2张为已知 (knownSlots[0], knownSlots[1] = true)
  ↓
4. 翻第一张到弃牌堆
   - drawCard(room)
   - discardCard(room, firstDiscard)
  ↓
5. sendGameStart(room) → 发送游戏开始通知
  ↓
6. sendTurnStart(room) → 发送回合开始通知
```

#### GameService.sendGameStart（GameService.cc:169-208）
```
for each player in room.players:
  构造 GameStartNotify:
    - room_id
    - round_number
    - first_player_id = room.players[0]->playerId
    - your_view (PlayerGameView):
      ├─ player_id
      ├─ own_cards: 自己的4张牌
      │   └─ 只有 is_known=true 的牌才暴露 value
      ├─ opponent_hands: 其他玩家的牌数量（不暴露数值）
      ├─ draw_pile: 抽牌堆数量
      ├─ discard_pile: 弃牌堆顶牌
      └─ scores: 所有玩家分数
  ↓
  sendToPlayer(player->conn, msg) → 个性化发送
```

**关键点：每个玩家收到的 `your_view` 内容不同！**
- 只能看到自己已知的牌
- 看不到其他玩家的牌值

#### GameService.sendTurnStart（GameService.cc:210-234）
```
1. turnNumber++
  ↓
2. 获取当前玩家
   current = room.players[room.currentPlayerSeat]
  ↓
3. 构造 TurnStartNotify:
   - room_id
   - current_player_id = current->playerId  ← 当前回合玩家ID
   - turn_number
   - round_number
   - phase (PLAYING / FINAL_ROUND)
   - draw_pile (数量)
   - discard_pile (顶牌)
   - final_round_remaining (如果是最终轮)
  ↓
4. broadcastToRoom(room, msg) → 广播给所有玩家
```

## 当前玩家判定总结

### 服务端权威判定
1. **存储**：`GameRoom.currentPlayerSeat` 存储当前回合玩家的座位号
2. **初始化**：`startNewRound()` 时设置为 0（第一个座位）
3. **推进**：`nextPlayer()` 时递增并循环
4. **验证**：所有操作请求都通过 `isCurrentPlayer()` 验证
5. **通知**：`TurnStartNotify` 广播当前玩家的 `playerId`

### 客户端被动接收
1. **接收**：从 `TurnStartNotify.current_player_id` 获取当前玩家ID
2. **存储**：保存到 `ProtoGateway.CurrentTurnPlayerId`
3. **判定**：UI层比较 `CurrentTurnPlayerId == myPlayerId`
4. **响应**：显示/隐藏操作按钮

### 关键规则
- ✅ 服务端是唯一权威来源
- ✅ 客户端只能被动接收，不能自行推进回合
- ✅ 座位号决定回合顺序（0→1→2→...→N-1→0）
- ✅ 最终轮会跳过稳态喊叫者
- ✅ 所有操作必须是当前玩家才能执行
