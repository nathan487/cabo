# 问题诊断：点击 Start Game 后界面不显示回合信息

## 🔍 问题现象

用户反馈：点击 Start Game 后，界面**没有显示**当前回合玩家的信息，也没有按钮显示。

## 🐛 根本原因分析

### 发现的问题

**问题位置：** `ProtoGateway.cs:468-482`

```csharp
private void OnGameStartNotify(GameStartNotify notify)
{
    Debug.Log($"[ProtoGateway] Game started: round={notify.RoundNumber}, firstPlayer={notify.FirstPlayerId}");
    GameSceneBootstrap.PendingGameStart = notify;  // ← 保存数据
    
    // 🚨 问题：会自动加载场景！
    if (coroutineOwner != null)
        coroutineOwner.StartCoroutine(LoadGameSceneNextFrame());
    
    OnStartGame?.Invoke(notify);  // ← 但此时可能还在旧场景
}

private System.Collections.IEnumerator LoadGameSceneNextFrame()
{
    yield return null; // wait one frame
    GameSceneBootstrap.LoadGameScene();  // ← 加载 GameScene
}
```

### 问题流程

```
1. 用户在 LobbyRoomScene 点击 Start Game
   ↓
2. 服务端发送 GameStartNotify
   ↓
3. ProtoGateway.OnGameStartNotify() 被调用
   ↓
4. 🚨 启动协程，下一帧加载 GameScene
   ↓
5. 触发 OnStartGame?.Invoke(notify) 
   ↓ (但此时仍在 LobbyRoomScene)
6. 【问题】GameTableUIToolkit 还不存在！
   ↓
7. 下一帧加载 GameScene
   ↓
8. GameSceneController.Start() 执行
   ↓
9. 找到 ProtoGateway，初始化 GameTableUIToolkit
   ↓
10. GameTableUIToolkit 订阅事件
   ↓
11. 检查 GameSceneBootstrap.PendingGameStart
    ↓
12. 调用 OnGameStarted(gs) ✅ (这个能收到)
    ↓
13. 服务端发送 TurnStartNotify
    ↓
14. 【问题可能点】此时订阅可能还没完成？
```

### 时序问题

**关键问题：** 有两个场景切换点

1. **接收到 GameStartNotify 时** → 还在 LobbyRoomScene
2. **加载 GameScene 后** → GameTableUIToolkit 才开始订阅事件

**潜在的竞态条件：**
- 如果服务端的 `TurnStartNotify` 在场景切换过程中发送
- 此时 GameTableUIToolkit 可能还没订阅 `OnTurnStart` 事件
- 导致 `OnTurnStarted()` 不会被调用

### 检查点 3：TurnStartNotify 的处理

**服务端发送：** `GameService.cc:165-166`
```cpp

```

**问题：**
- 两个消息几乎同时发送
- 客户端接收顺序：GameStartNotify → TurnStartNotify
- 但是 GameStartNotify 会触发场景切换
- **TurnStartNotify 到达时，可能还在切换场景中**

## 🚨 问题诊断结论

### 根本原因

**竞态条件（Race Condition）：**

```
时间线：
t0: 收到 GameStartNotify → 保存到 PendingGameStart
t1: 启动场景加载协程
t2: 触发 OnStartGame?.Invoke() (但 GameTableUIToolkit 不存在)
t3: 收到 TurnStartNotify → 触发 OnTurnStart?.Invoke()
    🚨 此时 GameTableUIToolkit 可能还没订阅！
t4: 下一帧，开始加载 GameScene
t5: GameSceneController.Start() 执行
t6: GameTableUIToolkit.Initialize() 执行
t7: 订阅 OnTurnStart 事件
    ❌ 但 TurnStartNotify 已经错过了！
```

### 为什么会这样？

1. **GameStartNotify 有 PendingGameStart 缓存机制** ✅
   - 即使订阅晚了，也能通过 PendingGameStart 手动调用

2. **TurnStartNotify 没有缓存机制** ❌
   - 如果订阅时已经发送过，就永远收不到了

3. **两个消息发送间隔极短**
   - 服务端几乎同时发送
   - 网络传输可能在 1-10ms 内都到达
   - 但场景切换需要至少 1-2 帧（16-32ms）

## 💡 解决方案

### 方案 1：为 TurnStartNotify 添加缓存机制（推荐）

**修改：** `ProtoGateway.cs`

```csharp
// 添加字段
private TurnStartNotify pendingTurnStart = null;

private void OnTurnStartNotify(TurnStartNotify notify)
{
    Debug.Log($"[ProtoGateway] ========== TurnStartNotify Received ==========");
    Debug.Log($"[ProtoGateway] Room: {notify.RoomId}");
    Debug.Log($"[ProtoGateway] Current Player: {notify.CurrentPlayerId}");
    Debug.Log($"[ProtoGateway] Turn: {notify.TurnNumber}");
    Debug.Log($"[ProtoGateway] Round: {notify.RoundNumber}");
    Debug.Log($"[ProtoGateway] Local Player ID: '{LocalPlayerId}'");
    Debug.Log($"[ProtoGateway] OnTurnStart subscribers: {(OnTurnStart?.GetInvocationList().Length ?? 0)}");
    
    CurrentTurnPlayerId = notify.CurrentPlayerId;
    CurrentRoundNumber = notify.RoundNumber;
    GameStateChanged?.Invoke();
    
    // 🔧 修复：如果没有订阅者，缓存这个通知
    if (OnTurnStart == null || OnTurnStart.GetInvocationList().Length == 0)
    {
        Debug.LogWarning("[ProtoGateway] ⚠️ No OnTurnStart subscribers yet, caching notification");
        pendingTurnStart = notify;
    }
    else
    {
        OnTurnStart?.Invoke(notify);
    }
}

// 添加公共方法供 GameTableUIToolkit 调用
public TurnStartNotify GetPendingTurnStart()
{
    var result = pendingTurnStart;
    pendingTurnStart = null;
    return result;
}
```

**修改：** `GameTableUIToolkit.cs`

```csharp
// 订阅游戏事件（仅在有 gateway 时）
if (gw != null)
{
    Debug.Log("[GameTableUIToolkit] Subscribing to gateway events...");
    gw.OnStartGame += OnGameStarted;
    gw.OnTurnStart += OnTurnStarted;
    // ... 其他订阅
    Debug.Log("[GameTableUIToolkit] ✅ All events subscribed");
    
    // 🔧 修复：检查是否有待处理的 TurnStartNotify
    var pendingTurn = gw.GetPendingTurnStart();
    if (pendingTurn != null)
    {
        Debug.Log("[GameTableUIToolkit] 📥 Processing pending TurnStartNotify");
        OnTurnStarted(pendingTurn);
    }
}
```

### 方案 2：延迟发送 TurnStartNotify（服务端修改）

**修改：** `GameService.cc`

```cpp

    // ... 初始化代码 ...
    
    sendGameStart(room);

    // 方案B：等待客户端发送 "SceneReady" 消息后再发送
    
    // 暂时不修改，优先用方案1
```

**缺点：** 需要修改服务端，增加复杂度

**修改：** `ProtoGateway.cs`

## 🎯 推荐方案

**推荐使用方案 1**：为 TurnStartNotify 添加缓存机制

**理由：**
1. ✅ 客户端修改，不影响服务端
2. ✅ 保持现有场景切换逻辑
3. ✅ 与 GameStartNotify 的 PendingGameStart 机制一致
4. ✅ 解决竞态条件问题
5. ✅ 对现有代码影响最小
