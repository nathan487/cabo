### 3. 服务端 - 当前玩家判定逻辑

#### ⚠️ isCurrentPlayer 函数存在潜在问题
**位置：** `GameService.cc:50-53`
```cpp
bool GameService::isCurrentPlayer(GameRoom& room, int64_t playerId) {
    auto p = getPlayer(room, playerId);
    return p && p->seatId == room.currentPlayerSeat;
}
```

**问题分析：**
这个函数**可能存在逻辑问题**，取决于 `PlayerGameState` 中的 `seatId` 是否与玩家在 `room.players` 数组中的**索引位置**一致。

#### 🔍 关键问题：seatId vs 数组索引

**场景1：玩家按加入顺序存储（正常情况）**
```cpp
room.players[0]->seatId = 0  ✅
room.players[1]->seatId = 1  ✅
room.players[2]->seatId = 2  ✅
room.players[3]->seatId = 3  ✅
```
此时 `isCurrentPlayer()` 工作正常。

**场景2：如果有玩家离开后重新加入（潜在问题）**
```cpp
// 初始状态
room.players = [
    {playerId: 10000, seatId: 0},
    {playerId: 10001, seatId: 1},
    {playerId: 10002, seatId: 2}
]

// 玩家 10001 离开
room.players = [
    {playerId: 10000, seatId: 0},
    {playerId: 10002, seatId: 2}  // ← 索引是1，但seatId是2
]

// currentPlayerSeat = 1 时
room.players[1]->seatId = 2  // ❌ 不匹配！
```

**但是，检查 `sendTurnStart` 的实现：**
```cpp
void GameService::sendTurnStart(GameRoom& room) {
    auto current = room.players[room.currentPlayerSeat];  // ← 直接用索引
    ts->set_current_player_id(current->playerId);
}
```

**这里也是直接用数组索引！**

#### 🚨 发现的不一致

有**两个不同的逻辑**在使用：

1. **isCurrentPlayer()**：比较 `p->seatId == room.currentPlayerSeat`
2. **sendTurnStart()**：直接用 `room.players[room.currentPlayerSeat]`

**问题：** 如果玩家列表发生变化（离开/重连），这两个逻辑可能不一致！

#### 💡 建议修复方案

**方案A：统一使用数组索引（推荐）**
```cpp
// 修改 isCurrentPlayer
bool GameService::isCurrentPlayer(GameRoom& room, int64_t playerId) {
    if (room.currentPlayerSeat < 0 || 
        room.currentPlayerSeat >= static_cast<int32_t>(room.players.size())) {
        return false;
    }
    return room.players[room.currentPlayerSeat]->playerId == playerId;
}

// 修改 nextPlayer
void GameService::nextPlayer(GameRoom& room) {
    room.currentPlayerSeat = (room.currentPlayerSeat + 1) % static_cast<int32_t>(room.players.size());
}
```

**方案B：确保 seatId 永远等于数组索引**
- 玩家加入时动态分配 seatId = 当前索引
- 玩家离开时重新整理 seatId
- 复杂度高，不推荐

#### 📝 当前实现的实际行为

由于当前代码**没有实现玩家离开游戏中的房间**的功能（只有房间阶段可以离开），所以：
- ✅ **游戏开始后玩家列表不变**
- ✅ **seatId 永远等于数组索引**
- ✅ **当前逻辑可以工作**

**但是：**
- ⚠️ 如果未来添加断线重连、中途退出等功能，需要修复这个逻辑
- ⚠️ 代码不够健壮，容易出 bug

```cpp
void GameService::sendTurnStart(GameRoom& room) {
    room.turnNumber++;
    auto current = room.players[room.currentPlayerSeat];  // 通过索引获取
    
    ::game::messages::ServerMessage msg;
    auto* ts = msg.mutable_turn_start_notify();
    ts->set_current_player_id(current->playerId);  // ← 发送 playerId
    ts->set_turn_number(room.turnNumber);
    ts->set_round_number(room.roundNumber);
    ts->set_phase(room.steadyCallerSeat >= 0
        ? ::game::common::GAME_PHASE_FINAL_ROUND 
        : ::game::common::GAME_PHASE_PLAYING);
    
    // 设置牌堆信息
    auto* dp = ts->mutable_draw_pile();
    dp->set_count(static_cast<int32_t>(room.drawPile.size()));
    auto* dcp = ts->mutable_discard_pile();
    dcp->set_count(static_cast<int32_t>(room.discardPile.size()));
    if (!room.discardPile.empty())
        *dcp->mutable_top_card() = room.discardPile.back();
    
    if (room.steadyCallerSeat >= 0)
        ts->set_final_round_remaining(room.finalRoundRemaining);
    
    broadcastToRoom(room, msg);  // 广播给所有玩家
}
```

**验证：** ✅ 正确
- ✅ 从 `room.players[currentPlayerSeat]` 获取当前玩家
- ✅ 发送的是 `playerId`（不是 seatId）
- ✅ 广播给所有玩家
- ✅ 包含完整的牌堆信息

```cpp
// 统一使用数组索引

        room.currentPlayerSeat >= static_cast<int32_t>(room.players.size())) {

}
```
