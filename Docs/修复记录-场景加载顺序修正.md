# 修复记录：场景加载顺序修正

## 📅 修复日期
2026-06-04

## 🐛 问题描述

点击 Start Game 后：
1. 看到一个一闪而过的 "YOUR TURN" 画面
2. 然后界面显示 "UI Toolkit Test Mode"
3. 进入独立测试模式，没有正确的游戏数据

## 🔍 根本原因

**处理顺序错误**：代码先检查 `TurnStartNotify`，后检查 `GameStartNotify`。

正确的顺序应该是：
1. **先处理 GameStartNotify** - 初始化游戏状态（牌面、分数等）
2. **再处理 TurnStartNotify** - 显示回合按钮

## ✅ 修复内容

### 修改 1：调整处理顺序 - GameTableUIToolkit.cs

**修改前（错误顺序）：**
```csharp
// 1. 订阅事件
// 2. 处理 TurnStartNotify  ← 先处理 Turn
// 3. 处理 GameStartNotify  ← 后处理 Game（错误！）
```

**修改后（正确顺序）：**
```csharp
// 1. 订阅事件
// 2. 先处理 GameStartNotify - 初始化游戏状态
// 3. 再处理 TurnStartNotify - 显示按钮
```

### 修改 2：移除独立测试模式

不再支持独立测试模式，如果没有 PendingGameStart 直接报错：

```csharp
if (gs != null)
{
    // 处理游戏数据
}
else
{
    Debug.LogError("❌ PendingGameStart is null! Cannot initialize game UI.");
    return;  // 提前退出
}
```

### 修改 3：注释掉 OnStartGame 事件触发

**ProtoGateway.cs:**
```csharp
private void OnGameStartNotify(GameStartNotify notify)
{
    GameSceneBootstrap.PendingGameStart = notify;
    coroutineOwner.StartCoroutine(LoadGameSceneNextFrame());
    
    // 不在这里触发事件，避免在错误场景处理
    // OnStartGame?.Invoke(notify);
}
```

### 修改 4：移除 GameSceneController 的独立模式

不再支持独立运行 GameScene，必须从 MainMenu → Lobby → Start Game 流程进入。

## 🎯 修复后的流程

```
1. 用户点击 Start Game
   ↓
2. 收到 GameStartNotify
   → 保存到 PendingGameStart
   → 启动场景加载
   ↓
3. 收到 TurnStartNotify
   → 缓存到 pendingTurnStart
   ↓
4. 场景加载完成
   ↓
5. GameTableUIToolkit.Initialize()
   ① 订阅事件
   ② 处理 PendingGameStart ✅
      - 显示牌面
      - 显示玩家信息
   ③ 处理 pendingTurnStart ✅
      - 显示按钮（如果是当前玩家）
      - 显示提示文本
   ↓
6. ✅ 界面正确显示
```
