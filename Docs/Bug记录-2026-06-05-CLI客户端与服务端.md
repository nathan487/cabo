## 🔴 严重 Bug

### BUG-1: 弃牌堆卡牌丢失

**文件**: `MuduoBaseGameServer/src/game/GameService.cc`
**行号**: 688-771（`handleTakeFromDiscard` 函数）

**问题描述**:
`handleTakeFromDiscard` 中，从弃牌堆弹出卡牌后，如果 `slot_indices` 为空或无效索引，函数直接 return，导致：
1. 卡牌永久从弃牌堆消失
2. 客户端不会收到任何响应，永久等待

**代码位置**:
- 行 699-700: `auto card = room->discardPile.back(); room->discardPile.pop_back();`
- 行 710: `if (indices.size() == 0) return;` — 卡牌已丢失
- 行 719-720: `if (slot < 0 || slot >= ...) return;` — 同样卡牌已丢失

**修复**: 提前验证，在 pop 之前检查所有条件；无效时发送错误响应。

### BUG-2: 非当前玩家可以劫持他人回合

**文件**: `MuduoBaseGameServer/src/game/GameService.cc`
**行号**: 553-594（`handleDiscardDrawn`）、596-686（`handleReplaceWithDrawn`）

**问题描述**:
两个函数都缺少 `isCurrentPlayer()` 检查。当玩家A抽牌后（step=WaitingDrawDecision），玩家B可以发送 DiscardDrawnReq 或 ReplaceWithDrawnReq，使用A的抽牌结果操作B自己的手牌。

**代码位置**:
- `handleDiscardDrawn`（行 560-561）: 只检查 `getPlayer` 非空，不检查是否当前玩家
- `handleReplaceWithDrawn`（行 603-604）: 同上

**修复**: 在两个函数中添加 `isCurrentPlayer(*room, req.player_id())` 检查。

---

### BUG-3: 网络缓冲区损坏后无法恢复

**文件**: `MuduoBaseGameServer/cli_client/src/NetworkClient.cpp`
**行号**: 208-226（`extractOneMessage` 函数）

**问题描述**:
`decodeFrame` 返回 false 时（如长度字段 > 10MB），`frameLen` 输出参数未设置，`extractOneMessage` 无法清理缓冲区。后续所有数据都堆积在损坏数据后面，连接彻底卡死。

**代码位置**:
- 行 212: `if (!decodeFrame(recvBuffer_, frameLen, payload)) { return false; }`
  - 此分支下 `frameLen` 未定义，无法 `erase`

**修复**: `decodeFrame` 返回 false 时，如果是因为长度 > 10MB（非法帧），清空整个缓冲区或至少清除头部无效数据。

## 🟠 与游戏规则不符的 Bug

### BUG-4: 对手卡牌数量不同步

**文件**: `MuduoBaseGameServer/src/game/GameService.cc`
**行号**: 267-297（`sendActionResult` 函数）

**问题描述**:
`sendActionResult` 构建的 `ActionResultNotify` 不包含各玩家的最新卡牌数量。对手的 `cardCount` 仅在 `GameStartNotify` 中设置一次。替换失败导致手牌增加（如4→5张）时，对手客户端无法感知。

**代码位置**:
- 行 275-297: `sendActionResult` 填充了 draw_pile、discard_pile、exchange_result，但没有玩家手牌数量

**修复**: 在 `ActionResultNotify` 或后续通知中添加玩家手牌数量更新。

### BUG-5: 初始看牌—玩家无法选择看哪两张

**文件**: `MuduoBaseGameServer/src/game/GameService.cc`
**行号**: 188-189

**规则原文**: "每位玩家可以秘密查看自己面前的**任意两张牌**"

**实际实现**:
```cpp
p->knownSlots[0] = true;
p->knownSlots[1] = true;
```
固定查看前两张。

**偏离**: 规则允许自由选择，实现固定 slot 0 和 1。

### BUG-6: 100分重置—自动执行而非玩家选择

**文件**: `MuduoBaseGameServer/src/game/GameService.cc`
**行号**: 421-428

**规则原文**: "如果某位玩家的累计分数恰好达到100分，**可以**将分数重置为50分"

**实际实现**:
```cpp
if (p->totalScore == 100 && !p->hasUsedReset) {
    p->totalScore = 50;  // 自动执行，不给玩家选择
    p->hasUsedReset = true;
}
```

**偏离**: "可以"暗示可选，代码自动执行。且客户端未处理 `hundred_triggers` 通知。

**相关客户端**: `cli_client/src/GameState.cpp` 行 568-589，ScoreUpdateNotify 处理忽略 `hundred_triggers`。
