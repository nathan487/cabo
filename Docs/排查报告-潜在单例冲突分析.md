### ⚠️ 需要关注的潜在风险

#### 1. ClientAppBootstrap的跨场景重复创建风险

**风险场景：**
如果用户在GameScene直接运行（不从MainMenu进入），会发生什么？

**当前代码分析：**
```csharp
// GameSceneController.cs
var bootstrap = FindObjectOfType<ClientAppBootstrap>();
if (bootstrap == null)
{
    Debug.LogError("❌ ClientAppBootstrap not found!");
    return;
}
```

**现状：**
- ✅ 代码已经有检查，如果找不到会报错并返回
- ✅ 没有自动创建逻辑
- ✅ **防御措施到位**

**建议：**
- 为ClientAppBootstrap也添加单例保护（类似GameSceneController）
- 防止意外在多个场景中放置

#### 2. RoomClientController和GameClientController的重复附加

**风险场景：**
这两个组件是通过 `gameObject.AddComponent<>()` 动态添加的：

```csharp
// ClientAppBootstrap.cs
var roomController = GetComponent<RoomClientController>();
if (roomController == null)
{
    roomController = gameObject.AddComponent<RoomClientController>();
}

if (GetComponent<Cabo.Client.Game.GameClientController>() == null)
{
    gameObject.AddComponent<Cabo.Client.Game.GameClientController>();
}
```

**分析：**
- ✅ 代码已经有 `GetComponent()` 检查
- ✅ 只有不存在时才添加
- ✅ **防御措施到位**

**但是：**
- 如果场景中手动放置了这些组件，会有两个实例吗？
- 需要检查场景文件

#### 3. FindObjectOfType的使用风险

**风险：**
如果场景中有重复组件，`FindObjectOfType<T>()` 只返回第一个找到的实例，可能导致：
- 找到错误的实例
- 不确定的行为
- 难以调试的问题

**当前使用情况：**
```csharp
// GameSceneController.cs
var bootstrap = FindObjectOfType<ClientAppBootstrap>();
var tableUIToolkit = FindObjectOfType<GameTableUIToolkit>();
```

**建议：**
- 对于应该是单例的组件，改用静态实例引用
- 或在Awake中添加重复检测

## 🔧 推荐的修复方案

### 优先级1：为ClientAppBootstrap添加单例保护

**原因：**
- 跨场景持久化组件
- 如果意外在多个场景放置，会导致严重问题
- 类似于GameSceneController的问题

**实现：**
```csharp
public sealed class ClientAppBootstrap : MonoBehaviour
{
    private static ClientAppBootstrap _instance;

    private void Awake()
    {
        // 单例检查
        if (_instance != null && _instance != this)
        {
            Debug.LogWarning($"[ClientAppBootstrap] ⚠️ Duplicate instance detected on '{gameObject.name}'. Destroying to prevent conflicts.");
            Destroy(gameObject);
            return;
        }

        _instance = this;
        DontDestroyOnLoad(gameObject);
        
        // ... 原有初始化代码
    }

    private void OnDestroy()
    {
        if (_instance == this)
        {
            _instance = null;
        }
    }
}
```

### 优先级2：为GameTableUIToolkit添加单例保护

**原因：**
- 场景级UI组件
- 如果场景中有重复，会导致：
  - 重复订阅事件
  - UI状态冲突
  - 按钮响应异常

**实现：**
类似GameSceneController的单例模式

### 优先级3：添加运行时单例验证

**目的：**
在开发阶段及时发现重复实例

**实现：**
```csharp
#if UNITY_EDITOR
private void OnValidate()
{

    if (instances.Length > 1)
```
