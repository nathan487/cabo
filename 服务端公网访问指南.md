# 服务端公网访问指南

## 启动服务端

```bash
cd MuduoBaseGameServer/build
LD_PRELOAD=$HOME/anaconda3/lib/libprotobuf.so.31 ./GameServer 8888
```

---

## 方案 A：局域网直连（最简单，同一 WiFi/网络下）

同一局域网的其他电脑直接连 WSL 的 IP：

```bash
# 查看 IP
hostname -I
# 输出类似: 172.24.165.161
```

客户端连接 `172.24.165.161:8888`

**如果连不上**：Windows 防火墙需开放端口（管理员 PowerShell）：
```powershell
New-NetFirewallRule -DisplayName "Cabo" -Direction Inbound -Port 8888 -Protocol TCP -Action Allow
```

---

## 方案 B：路由器端口转发（公网 IP，永久）

1. 登录路由器管理页面（通常是 192.168.1.1）
2. 找到"端口转发/Port Forwarding/Virtual Server"
3. 添加规则：外部端口 8888 → 内部 IP `172.24.165.161`（WSL IP）→ 内部端口 8888
4. 还需要在 Windows 防火墙开端口（同方案 A）
5. 查公网 IP：浏览器打开 https://ip.sb
6. 客户端连接 `你的公网IP:8888`

**注意**：家用宽带多半没有固定公网 IP（NAT 后）。如果 ip.sb 显示的 IP 和路由器 WAN 口 IP 不一致，说明运营商做了 NAT，此方案不可用。

---

## 方案 C：Tailscale 虚拟组网（免费，无需公网 IP）

所有玩家装 Tailscale，自动组成虚拟局域网：

1. 每台电脑去 https://tailscale.com/download 下载安装
2. 所有人登录同一个账号（或用分享功能）
3. 服务端主机在 Tailscale 里能看到一个 `100.x.x.x` 的 IP
4. 客户端连接 `100.x.x.x:8888`

**优点**：免费、无需公网 IP、无需改路由器、加密传输

---

## 方案 D：ngrok TCP（付费/绑卡）

ngrok 免费版 TCP 隧道需要绑定信用卡（不扣费）：

1. https://dashboard.ngrok.com/signup 注册
2. 设置 authtoken：`~/.local/bin/ngrok config add-authtoken 你的token`
3. 在 https://dashboard.ngrok.com/settings#id-verification 绑定卡片
4. 启动：`~/.local/bin/ngrok tcp 8888`
5. 把显示的 `0.tcp.ngrok.io:XXXXX` 发给客户端

---

## 推荐优先级

1. **同一局域网** → 方案 A（零配置）
2. **异地但要稳定** → 方案 C Tailscale（推荐！免费且简单）
3. **有公网 IP** → 方案 B
4. **愿意绑卡** → 方案 D
